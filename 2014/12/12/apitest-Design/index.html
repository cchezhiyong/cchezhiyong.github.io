接口测试框架2W1H

Why

为什么要弄这么一个框架？

  • 测试代码易读性差
  • 没有集成stub和mock
  • 数据库和NoSQL测试不够友好
  • 测试基类继承和重载功能不够完善

我设想的框架是这样子的：

What

Basic BDD framework

  • BDD Common
  • Request
  • Response assert
  • Setup
  • Teardown
  • Stub
  • Mock
  • DB assert
  • NoSQL assert
  • Extend and Overload
  • Exception
  • Log and Debug
  • Perf

Potal testcase manager

How

粉红色部分需要代码二次封装

  • Request and Response and assert and Setup and Teardown is supported by cucumber bdd basic framework.
  • Stub and Mock,we will use webmock.
  • Others will dev.
